## 使用方法

### 基本调用

在 Claude Code 中直接说：

```
请分析 docs/文献文件夹/ 目录下的所有文献，并生成课程
```

或使用命令格式：

```
# 分析文件夹中的多个文献
/extract-course docs/文献文件夹/

# 分析单个或多个文件（包括转录文件）
/extract-course 转录文件.md
/extract-course 文献文件.md
```

### 智能处理功能

命令会自动识别和处理不同类型的内容：

- **转录内容**：自动检测发言人、时间戳、配图等特征，生成总览和专题文档
- **普通文献**：标准文献分析逻辑，生成结构化课程文档
- **混合内容**：支持多种格式文件的综合分析

## Claude Code 执行流程

### 第1步：读取文件内容

根据输入类型读取文件：

- **文件夹输入**：使用 Glob 工具一次性读取指定目录下的所有文献文件（支持1-100+个文件）
- **单文件输入**：直接读取指定的 markdown 文件

### 第1.5步：智能识别内容特征

检测文件特征，自动识别内容类型：

- 发言人标记：发言人1、发言人2等
- 时间戳模式：00:00、12:34等格式
- 配图链接：包含图片URL链接
- 转录特征：口语化表达、现场演示描述等

根据识别的内容特征，自动选择最合适的处理方式。

### 第2步：内容分析与预处理

根据识别的内容特征，智能选择处理方式：

```
请分析以下内容，进行整体主题识别和内容分类。

要求：
1. **整体把握**：理解所有内容的整体脉络和逻辑关系
2. **主题识别**：基于内容自动识别主要主题
3. **内容优化**：
   - 识别并保留重要观点、具体例子、重要表述
   - 清洗冗余信息（如过度口语化表达、重复语句）
   - 提取配图信息并合理安排位置
   - 将口语化内容转换为专业书面语
4. **结构化组织**：按主题自动分组，标注各主题的核心观点
5. **完整提取**：保留关键观点、重要细节、具体案例、金句

分析格式：
===整体分析开始===
内容来源: [文件来源描述]
内容特征: [转录内容/普通文献/混合内容]

核心主题分类:
1. [主题A] - 核心观点概述
   - 关键信息要点
   - 相关内容位置

2. [主题B] - 核心观点概述
   - 关键信息要点
   - 相关内容位置

配图清单:
[如果有配图，列出图片链接和对应主题]

===整体分析结束===

完整内容：
[以下是所有内容的详细分析]
```

### 第3步：生成课程大纲

基于AI分析结果，生成完整课程大纲：

```
基于以下主题分析结果，整理系统化的课程大纲：

要求：
1. **课程命名**：基于内容主题，生成简洁课程名称
2. **章节设置**：根据分析结果设置章节，确保覆盖所有重要主题
3. **章节划分**：每个章节对应一个核心主题，体现内容的逻辑结构
4. **内容整合**：
   - 识别实践案例和应用场景，将其归入最相关的主题章节
   - 将未来发展趋势和展望内容分配到相应主题的结尾部分
   - 避免为案例或展望创建独立章节
5. **输出格式**：默认生成总览文档和各专题文档

课程大纲格式：
===课程大纲开始===
课程标题：[基于内容生成的课程名称]

第一章 [主题名称] - 核心观点摘要
   [相关案例和应用场景将融入本章节]
第二章 [主题名称] - 核心观点摘要
   [相关案例和应用场景将融入本章节]
第三章 [主题名称] - 核心观点摘要
   [相关案例和应用场景将融入本章节]

输出计划：
- 总览文档：课程概览和学习路径
- 专题文档：每个主题的详细内容（包含融合的案例和展望）
===课程大纲结束===
```

**专题文档示例（叙述性写作）：**
```
# 01_Markdown：AI时代的法律文本写作利器

## 引言
在人工智能浪潮席卷各行各业的今天，法律行业正面临着前所未有的变革与机遇。传统的文档处理方式已难以满足高效协作和智能处理的需求。在这样的背景下，Markdown作为一种轻量级标记语言，凭借其独特的优势，正成为法律从业者迎接AI时代的重要工具。

## 核心概念：为什么选择Markdown

Markdown的设计哲学体现了"简单即美"的理念。它通过最少的标记符号，实现了文本内容与格式的有效分离。这种设计不仅让文档编辑变得更加专注和高效，更重要的是，它完美契合了AI时代的文本处理需求。

当法律从业者开始使用Markdown时，会立即感受到它带来的变化。文档不再受限于特定软件的格式束缚，内容以纯文本形式存在，可以在任何平台、任何设备上打开和编辑。这种开放性和兼容性，为法律工作的数字化转型奠定了坚实的基础。

## Markdown在法律工作中的深度应用

### 知识管理体系的构建

在法律实务中，知识管理是提升工作效率和专业水平的关键。传统的Word文档虽然在格式表现上更为丰富，但在知识组织和检索方面却存在诸多不便。而Markdown的结构化特性，使得法律从业者可以轻松构建层次分明的知识库。

以合同范本库为例，使用Markdown可以通过标题层级清晰地组织不同类型的合同模板。律师们不仅可以快速定位所需模板，还能利用Markdown的链接功能，在相关条款间建立知识关联，形成完整的知识网络。

### 法律文书协作的新范式

法律工作往往涉及多人协作，传统的文档交换方式容易出现版本混乱、修改痕迹难以追踪等问题。Markdown配合Git等版本控制工具，为法律文书协作带来了革命性的改变。

当团队成员对一份法律意见书进行修改时，每一次变更都被清晰地记录下来。谁在何时做了什么修改，修改了哪些内容，这些信息都一目了然。更重要的是，由于Markdown是纯文本格式，即使出现版本冲突，也能够通过简单的文本对比快速解决问题。

### AI协作的无缝对接

随着ChatGPT、Claude等AI工具的普及，法律从业者越来越多地借助AI来提升工作效率。Markdown的纯文本特性使其成为与AI协作的理想媒介。

当律师需要AI辅助分析一份合同时，可以直接将Markdown格式的合同内容发送给AI。由于内容结构清晰、格式简单，AI能够准确理解合同条款的逻辑关系，提供更有价值的分析结果。同样，当AI生成法律文书初稿时，以Markdown格式返回的内容也更便于律师进行后续修改和完善。

## 实践案例：从理论到应用

张律师是国内某知名律所的合伙人，他最早在自己的团队中推广使用Markdown。起初，团队成员对这种新的写作方式持保留态度，但在实际应用中，他们逐渐发现了Markdown的巨大价值。

在一起复杂的并购案中，张律师的团队需要整理和分析上百份相关文件。他们将所有文件转换为Markdown格式，利用标签和链接功能，构建了一个完整的案件知识图谱。通过这种方式，团队成员可以快速定位关键信息，大大提升了工作效率。
```

### 专题文档生成说明

**重要：实践案例和展望内容的整合原则**

当分析内容包含实践案例、应用场景或未来展望时，应智能地将其融入相应的主题专题中，而不是创建独立的章节。

**整合策略：**
1. **案例匹配**：将案例内容按照主题相关性分配到对应的专题文档中
2. **内容融合**：在专题文档的"实践案例分析"部分自然地融入相关案例
3. **展望分配**：将未来发展趋势的内容融入相关主题的结尾部分
4. **智能判断**：AI应根据内容的核心主题和应用领域进行合理分配

- 在第3步生成课程大纲时，识别案例和展望内容的主题归属
