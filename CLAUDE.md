# AI 执行约束

## 核心规则

### Skill 调用

当用户请求匹配某个 Skill 时，**必须先使用 `Skill` 工具**调用该 skill，再执行任务。

### 输出规范

CLI 命令产生的所有命名产出（报告、日志、结果等），**必须写入输入文件所在目录的目标文件**，而非仅打印到终端。

文件名格式：`YYMMDD 原始文件名.扩展名`（日期为执行当天的日期）

| 输入 | 输出 |
| --- | --- |
| `audio.wav` | `260410 audio.docx` |
| `data.json` | `260410 data.md` |
| `合同.pdf` | `260410 合同.md` |

### 错误处理

- 可恢复错误：重试或使用备用方案
- 不可恢复错误：记录到日志，向用户返回清晰错误信息
- 敏感操作（删除、覆盖）：先确认再执行

### Raycast 扩展重建与重载

修改代码后需要重新构建并重载扩展：

1. **构建**：`npm run build`（在项目目录）
2. **重载**：`npm run dev`（后台执行即可触发 Raycast 自动重载，无需手动操作）

> 执行方式：`npm run dev & sleep 5 && kill %1` — 启动开发模式等待几秒让 Raycast 识别变更后自动终止进程。
